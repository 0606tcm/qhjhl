## 一、需求分析

### 1.1 场景背景

资管公司需要一个内部 Web 工具，帮助渠道销售团队管理基金产品信息和客户关系。

### 1.2 用户画像

| 角色 | 核心职责 | 主要诉求 |
|------|----------|----------|
| **渠道销售** | 服务客户、推广产品 | 快速查产品、记录跟进、了解客户持仓 |
| **团队长** | 管理团队、把控业绩 | 查看团队排行、监控整体业务数据 |

### 1.3 核心痛点与解决方案

| 痛点 | 解决方案 |
|------|----------|
| 产品信息分散，查找困难 | 统一产品货架 + 智能搜索 |
| 客户跟进记录易遗漏 | 结构化跟进记录 + AI 辅助录入 |
| 难以快速回答"谁买了什么" | 双向关联查询（产品→客户、客户→产品） |
| 缺乏业务全局视角 | 可视化 Dashboard |
| 团队长难以了解团队情况 | 团队排行榜 + 综合评分 |

### 1.4 功能优先级

| 优先级 | 功能 |
|--------|------|
| P0 (必做) | 产品列表/筛选/详情、客户列表/详情/持仓、跟进记录查看/新增、Dashboard 核心图表 |
| P1 (应做) | 新增客户、角色视角切换、团队排行、服务统计 |
| P2 (加分) | AI Agent 三能力（查询/图表/录入） |

## 三、UI 设计规范

### 3.1 设计原则

| 原则 | 说明 | 实践 |
|------|------|------|
| **效率优先** | 减少操作步骤，信息触手可及 | 侧边栏详情、快捷筛选、全局搜索 |
| **清晰层级** | 重要信息突出，次要信息收敛 | 卡片分组、渐进式披露 |
| **专业可信** | 符合金融行业调性，传递信任感 | 金色调、规整布局、数据精确 |
| **一致体验** | 相同场景采用相同交互模式 | 统一组件、统一反馈 |

**设计关键词**：`专业 · 高效 · 清晰 · 尊贵`

#### 金色使用原则

```
✓ 适合使用金色：
  • 主要 CTA 按钮
  • Logo 和品牌元素
  • 导航选中态
  • 重要数据强调（如 AUM 总额）
  • 图标高亮
  • 选中/激活状态的边框

✗ 避免使用金色：
  • 大面积背景（会显得浮夸）
  • 普通正文文字
  • 错误/警告状态（用语义色）
  • 次要元素

金色占比建议：页面整体 5-10%
```
